### (1) local network interface 설정
```
- 송신, 수신 host 모두 동일하게 설정(local network name은 달라도 상관 없음)
- local network name: 20111.NDV.UT.B.KC(이런 형식으로 입력)
- Service 20111
- Network Specification ;230.2.1.11;
- import,export subject(RVPERF.>, _RVCM.>, TEST.>) 
```

### (2) local network interface 설정 + 테스트(1개 보냈는데, 3개가 출력됨.)
```
C:\Users\Administrator>tibrvsend -service 20111 -network ";230.2.1.11;" -daemon tcp:9201 "TEST.TOPIC" "Hello RVRD"
Publishing: subject=TEST.TOPIC "Hello RVRD"

C:\Users\Administrator>tibrvlisten -service 20111 -network ";230.2.1.11" -daemon tcp:9201 "TEST.>"
tibrvlisten: Listening to subject TEST.>

### (5) 3개 출력될 때 wireshark 확인 
```

No.1  rvrd-B → 멀티캐스트 (최초발행)     → tibrvlisten 1번째 수신
No.3  rvrd-A → rvrd-B TCP 포워딩         (역방향)
No.4  rvrd-B → 멀티캐스트 재발행         → tibrvlisten 2번째 수신
No.5  rvrd-B → rvrd-A TCP 포워딩         (정방향)
No.6  rvrd-A → 멀티캐스트 재발행         → tibrvlisten 3번째 수신
No.8  rvrd-A → rvrd-B ACK               (수신확인)          

tibrvsend → rvrd-B → No.1 rvrd-B 멀티캐스트 배포 → tibrvlisten 1번째 수신
No.1 rvrd-B 멀티캐스트 배포 → rvrd-A → rvrd-B TCP 포워딩 → rvrd-B → No.4 rvrd-B 멀티캐스트 배포 → tibrvlisten 수신 2번째
No.4 rvrd-B 멀티캐스트 배포 → rvrd-B 본인 수신 → rvrd-A TCP 포워딩 → No.6 rvrd-A 멀티캐스트 배포 → tibrvlisten 수신 3번째
